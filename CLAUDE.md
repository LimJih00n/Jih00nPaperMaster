# 논문 분석 마스터 프롬프트

## 🚀 AI 증강 독서 3단계 워크플로우

### 🔍 1단계: 정찰 드론 모드 (탐색 및 필터링)
**목표**: 읽을 가치가 있는 논문을 10배 빠르게 선별

#### 실행 명령:
```
"papers/ 폴더의 모든 논문을 스캔하고 각각 한 줄 요약해줘"
"[논문1.pdf, 논문2.pdf, 논문3.pdf]의 핵심 기여와 방법론만 간단히 비교해줘"
"이 중에서 [특정 주제]와 가장 관련 있는 논문 2개를 추천해줘"
```

#### 정찰 보고서 형식:
각 논문마다:
- **한 문장 요약**: 이 논문이 해결하려는 핵심 문제
- **핵심 기여**: 새롭게 밝혀내거나 만들어낸 것
- **방법론**: 사용한 주요 기법 (예: Transformer, GAN, etc.)
- **관련성 점수**: 사용자 관심사와의 관련성 (1-5점)

### 🔬 2단계: 전문 분석팀 모드 (해부 및 심문)
**목표**: 선정된 논문의 뼈대와 논리를 완벽하게 분해

#### 구조적 분석 명령:
```
"[논문명.pdf]를 아래 구조로 상세 분석해줘:
1. 연구 배경 및 문제 정의
2. 핵심 가설 또는 주장
3. 실험 설계 (단계별)
4. 주요 결과와 의미
5. 저자의 결론 및 한계점"
```

#### 심층 심문 질문 템플릿:
```
"Figure [X]가 정확히 뭘 의미하는지 초등학생도 이해하게 설명해줘"
"왜 [방법A]가 아니라 [방법B]를 선택했을까? 장단점 비교해줘"
"Table [Y]의 결과가 통계적으로 유의미한가? 근거는?"
"저자가 놓친 실험이나 분석이 있다면?"
"이 주장의 가장 약한 부분은 어디야?"
```

#### 🔍 필수 검색 작업:
```
"이 논문을 인용한 최신 논문 5개 찾아서 어떻게 평가하는지 알려줘"
"arXiv에서 이 논문의 개선 버전이나 후속 논문 검색해줘"
"이 논문의 저자들의 다른 연구도 검색해서 연구 방향 파악해줘"
"이 분야의 최신 트렌드와 이 논문의 위치를 검색해서 알려줘"
"이 논문의 방법론을 실제로 구현한 GitHub 코드 검색해줘"
"이 논문과 상반된 주장의 논문들을 검색해서 비교해줘"
"arXiv에서 이 주제의 2024-2025 최신 프리프린트 찾아줘"
```

### 💡 3단계: 스파링 파트너 모드 (연결 및 창조)
**목표**: 새로운 아이디어 창출과 내 연구와의 연결

#### 창의적 확장 명령:
```
"이 논문의 잠재적 허점 3가지와 개선 방안 제시해줘"
"이 방법론을 [내 연구 분야]에 적용하는 구체적 아이디어 5개"
"이 논문과 상반된 결론의 연구를 찾아 비교 분석해줘"
"이 논문 이후 5년 뒤 연구는 어떻게 발전할까?"
"이 논문의 아이디어로 스타트업을 만든다면?"
```

#### 🌐 연결 및 확장 검색:
```
"이 논문과 유사한 아이디어를 다른 분야에 적용한 사례 검색해줘"
"이 기술의 최신 상용화 사례와 기업들 검색해줘"
"관련 특허나 IP 정보 검색해서 산업 동향 파악해줘"
"이 분야 Top 연구자들과 최신 연구 그룹 검색해줘"
"이 논문 이후 breakthrough 연구들 검색해서 발전 방향 분석해줘"
```

## 📚 표준 논문 분석 프로세스

### 1. 논문 선택 및 읽기
사용자가 `papers/` 폴더에서 논문을 지정하면:
1. 3단계 워크플로우에 따라 체계적으로 분석
2. `outcome/[논문제목]/` 폴더 생성
3. 단계별 분석 파일 생성:
   - `scout_report.md`: 1단계 정찰 보고서
   - `deep_analysis.md`: 2단계 심층 분석
   - `creative_insights.md`: 3단계 창의적 통찰
   - `summary.md`: 종합 요약

### 2. 논문 분석 구조

#### 📄 scout_report.md (1단계 정찰 보고서)
```markdown
# [논문 제목] - 정찰 보고서

## 🎯 Quick Scan
- **한 줄 요약**: [이 논문이 해결하려는 핵심 문제]
- **핵심 기여**: [새롭게 제시한 방법/발견]
- **방법론**: [주요 기술/알고리즘]
- **실험 규모**: [데이터셋 크기, 실험 수]
- **성능 향상**: [주요 지표에서의 개선율]
- **관련성 점수**: [1-5점] - [평가 이유]

## ⚡ 읽을 가치 판단
- **강점**: [눈에 띄는 장점 2-3개]
- **약점**: [명백한 한계 2-3개]
- **추천 여부**: [정독 추천/스킵/참고용]
```

```markdown
- **재현성 검증**: GitHub에서 구현 코드 검색 및 확인
```

### 3. 분석 시 준수 사항

#### 🔍 필수 검색 체크리스트:
1. **논문 검증**: Google Scholar에서 인용 수와 평가 검색
2. **최신 프리프린트**: arXiv에서 관련 최신 논문 및 개선 버전 검색
3. **코드 확인**: GitHub에서 구현 코드 검색
4. **최신 동향**: 2024-2025년 관련 최신 논문 검색 (arXiv 우선)
5. **산업 적용**: 실제 사용 사례와 기업 활용 검색
6. **연구자 추적**: 저자의 후속 연구 및 연구 그룹 검색

#### 깊이 있는 분석을 위해:
1. **비판적 사고**: 저자의 주장을 그대로 받아들이지 말고 검증
2. **맥락 이해**: 이 논문이 나온 시기와 배경 고려
3. **재현 가능성**: 실제로 구현 가능한지 평가
4. **실용성**: 현실 문제에 적용 가능한지 판단

#### 명확한 설명을 위해:
1. **전문용어 설명**: 처음 나오는 용어는 반드시 쉽게 풀어서 설명
2. **예시 활용**: 복잡한 개념은 구체적 예시로 설명
3. **시각적 표현**: 필요시 다이어그램을 텍스트로 표현
4. **단계별 설명**: 복잡한 과정은 step-by-step으로 분해

#### 날카로운 분석을 위해:
1. **가정 검토**: 논문의 전제 조건과 가정이 현실적인지
2. **실험 설계 평가**: 실험이 주장을 충분히 뒷받침하는지
3. **일반화 가능성**: 다른 도메인에도 적용 가능한지
4. **계산 복잡도**: 실제 스케일에서 작동 가능한지

### 4. 추가 산출물

필요시 생성할 추가 파일들:
- `outcome/[논문제목]/implementation_notes.md`: 구현 관련 메모
- `outcome/[논문제목]/questions.md`: 추가 학습이 필요한 질문들
- `outcome/[논문제목]/related_papers.md`: 읽어야 할 관련 논문 리스트

### 5. 단계별 실행 명령 모음

#### 🔍 1단계 명령어 (정찰 + 검색)
```
"papers/ 폴더의 모든 논문 스캔하고 한 줄씩 요약해줘"
"[논문1.pdf, 논문2.pdf]를 빠르게 스캔하고 정찰 보고서 작성해줘"
"이 중 [transformer/GAN/RL] 관련 논문만 추려서 비교해줘"
"가장 영향력 있을 것 같은 논문 2개 추천해줘"
"각 논문의 인용 수를 Google Scholar에서 검색해서 알려줘"
"arXiv에서 이 논문들의 최신 버전이나 개선판 있는지 확인해줘"
"이 논문들 중 2024년에 가장 주목받은 논문 검색해줘"
```

#### 🔬 2단계 명령어 (분석 + 검색)

## 🛠️ 유용한 팁

### 효율적인 논문 읽기 순서
1. **Abstract + Conclusion** → 5분 정찰
2. **Figures + Tables** → 시각적 이해
3. **Introduction + Related Work** → 맥락 파악
4. **Methodology** → 핵심 방법 이해
5. **Experiments** → 검증 과정 확인
6. **Discussion** → 깊은 통찰 획득

### 논문 읽기 체크리스트
- [ ] 이 논문이 해결하려는 진짜 문제는?
- [ ] 기존 방법과 뭐가 다른가?
- [ ] 실험이 주장을 충분히 뒷받침하는가?
- [ ] 재현 가능한가?
- [ ] 내 연구에 어떻게 활용할까?
- [ ] 이 논문의 약점은?
- [ ] 후속 연구 아이디어는?

### 위험 신호 (Red Flags)
⚠️ 과도한 주장 without 충분한 근거
⚠️ Cherry-picked 실험 결과
⚠️ 중요한 baseline 비교 누락
⚠️ 재현 불가능한 실험 설정
⚠️ 통계적 유의성 미제시
⚠️ 한계점 논의 부재

## 📝 마지막 조언
- **첫 읽기는 빠르게**: 완벽하게 이해하려 하지 말고 전체 그림 파악
